## 🎯 Введение
Традиционные лексические методы поиска (например, BM25, TF-IDF) в исходном коде и технической документации часто терпят неудачу, когда разработчики используют запросы на естественном языке, которые не совсем соответствуют идентификаторам кода. **Семантический поиск кода (Semantic Code Retrieval)** решает эту проблему путем проецирования исходного кода и запросов в общее многомерное векторное пространство (эмбеддинги), что позволяет извлекать релевантный код на основе его *логического назначения*, а а не точного совпадения ключевых слов.

---

## 📂 Цель проекта и набор данных

Целью данного эксперимента была разработка конвейера векторизации и оценки качества поиска на базе Python 3.12 в среде Jupyter Notebook, проведение сравнительного анализа трех моделей-трансформеров и визуальное обоснование выбора лучшего решения.

Проект использует набор данных, состоящий из трех основных артефактов:
*   `code_corpus.json` — Корпус из 200 многоязычных функций (Python и Java), каждая с сигнатурой, исходным кодом, русской документацией и ключом категории.
*   `eval_questions.json` — 25 контрольных многоязычных вопросов (на русском и английском языках) с известными эталонными идентификаторами функций (`correct_chunk_id`).
*   `categories.json` — Справочник из 5 функциональных доменов разработки: `auth`, `database`, `http`, `validation`, `utils`.

## 🧠 Архитектура решения и выбор моделей

Для обеспечения максимального семантического покрытия была применена **стратегия конкатенации текстовых признаков**. Для каждой функции в корпусе три метаполя (название метода, описание docstring и исходный код) были объединены для формирования ее эмбеддинга: `f"{function_name} {description} {code}"`.

Поиск осуществлялся путем вычисления косинусного сходства между вектором запроса $\mathbf{q}$ и матрицей корпуса $\mathbf{C}$:

$$\text{sim}(\mathbf{q}, \mathbf{c}_i) = \frac{\mathbf{q} \cdot \mathbf{c}_i}{\|\mathbf{q}\| \|\mathbf{c}_i\|}$$

Были выбраны и сравнены три архитектуры нейронных сетей:

1.  **`paraphrase-multilingual-MiniLM-L12-v2`** *(Базовый кандидат)*: Легковесная, многоязычная модель дистиллированного трансформера (384 измерения, ~118 миллионов параметров), оптимизированная для высокой скорости вывода и низкого потребления памяти.
2.  **`paraphrase-multilingual-mpnet-base-v2`** *(Продвинутый кандидат)*: Более надежная модель на основе MPNet (768 измерений, ~278 миллионов параметров), сочетающая преимущества авторегрессионного моделирования (XLNet) и автокодирования (BERT).
3.  **`sentence-transformers/LaBSE`** *(Альтернативный кандидат для дополнительной задачи)*: LaBSE (Language-agnostic BERT Sentence Embedding) от Google, обученная на выравнивание параллельных предложений из 109 языков. Используется для проверки гипотезы о превосходстве кросс-языкового переноса.

## 📊 Результаты экспериментов

Качество системы поиска оценивалось с использованием метрики **Precision@3**, которая представляет собой долю из 25 запросов, для которых эталонный код попал в топ-3 релевантных результатов.

| Модель векторизации | Размерность вектора | Время обработки (200 функций) | Precision@3 | Статус решения |
| :--- | :---: | :---: | :---: | :---: |
| **`paraphrase-multilingual-mpnet-base-v2`** | **768** | **~14.2 сек** | **0.8800** | 🏆 **Оптимальный выбор** |
| `paraphrase-multilingual-MiniLM-L12-v2` | 384 | ~4.8 сек | 0.8400 | ⚡ Высокоскоростная альтернатива |
| `sentence-transformers/LaBSE` | 768 | ~22.1 сек | 0.6400 | ❌ Не соответствует качеству |

## 📈 Анализ и визуализации

### 1. Языковой разрез и гистограмма ошибок (расширенная аналитика)
Расширенный модуль `evaluate_model_detailed` был разработан для декомпозиции метрических компонентов лучшей модели (`mpnet-base-v2`) и их визуализации. Модель продемонстрировала стабильно высокую производительность как для английских (90,0%), так и для русских (86,7%) запросов, доказав свою способность преодолевать языковой барьер. Ошибки были равномерно распределены по категориям `auth`, `database` и `utils`, часто возникая при коротких, абстрактных запросах.

### 2. Двумерная проекция семантического пространства (t-SNE)
768-мерное пространство эмбеддингов было сокращено до 2 координат с использованием алгоритма **t-SNE** для визуализации. Эта визуализация наглядно показала 5 изолированных кластеров, соответствующих функциональным доменам: `database`, `validation`, `auth`, `http` и `utils`. Эта сегрегация указывает на то, что трансформер эффективно абстрагируется от языка программирования (Python или Java) и кластеризует функции на основе их *функциональной бизнес-логики*, что крайне важно для надежного семантического поиска.

## 🎓 Заключение

На основании комплексной оценки **`paraphrase-multilingual-mpnet-base-v2`** признана бесспорным лидером с итоговым показателем **Precision@3 = 0.8800**.

**Рекомендации для промышленного внедрения:** Модель `paraphrase-multilingual-mpnet-base-v2` рекомендуется благодаря ее продвинутой архитектуре (278 миллионов параметров), обеспечивающей глубокое понимание сложного многоязычного контекста на стыке естественного языка и синтаксиса программирования. Она обеспечивает высокую точность как для русских (86,7%), так и для английских (90,0%) запросов. Для систем с критическими ограничениями вычислительных ресурсов и задержки (latency) можно рассмотреть `MiniLM-L12-v2` как более быструю альтернативу с потерей точности всего на 4%.

### 🚀 Настройка проекта локально

Требуется **Python версии 3.12** или выше.

### Шаг 1. Клонирование репозитория
Откройте терминал (или Git Bash) и клонируйте проект на свой компьютер:
```bash
git clone https://github.com/HazDeen/semantic-code-search-mtuci.git
cd semantic-code-search-mtuci
```

### Шаг 2. Установка зависимостей
Рекомендуется создать виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 3. Запуск Jupyter Notebook
Откройте `solution.ipynb` в VS Code или Jupyter Lab, чтобы изучить код, запустить эксперименты и воспроизвести результаты.
